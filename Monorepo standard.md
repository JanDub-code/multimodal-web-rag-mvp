# Monorepo standard

Tento dokument je referenční vstup.

Závazná pravidla pro strukturu repozitáře, migration triggers a roadmapu jsou sjednocené v:
- `Architektura systému pro lokální Docker provoz.md`

## Důvod

Repo má mít jednu pravdu pro:
- runtime architekturu,
- pravidla MVP baseline vs scale target,
- pořadí implementačních kroků.

Tímto se předchází driftu mezi „architektura“ a „monorepo standard“ dokumentací.
